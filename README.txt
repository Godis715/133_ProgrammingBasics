#133_ProgrammingBasics

Векторный графический редактор
Основными объектами являются геометрические примитивы и требования - структуры,
задающие отношения между примитивами

Программа должна состоять из различных модулей, каждый из которых отвечает за исполнение
определенных обязатенностей

Что у нас есть:
        1) Прописаны основные классы примитивов и реализована работа с ними
        2) Прописаны классы требований и реализована работа с ними
        3) Реализован механизм поддержки выполнения требований, наложенных на объекты сцены
        4) Базовый графический интерфейс

Что планируется сделать:
        1) Сохранение, загрузка сцены (Дмитрий Терещенко)
        2) Реализация скриптового языка для взаимодействия с редактором (Евгений Олешкевич)
        3) Улучшение графического интерфейса (Дмитрий Мельников, Дмитрий Терещенко)
        4) Рефакторинг и внесение изменений в архитекутуру программы (Игнатьев Денис)
        5) Добавить систему логгирования и обработки исключений (Игнатьев Денис)
        6) Оптимизация программы (Денис Игнатьев, Евгений Олешкевич)
        7) Написание тестовой системы (Дмитрий Мельников)
        8) Механизм отображения требований и работы с ним через пользовательский интерфейс (Дмитрий Терещенко, Евгений Олешкевич)
        9) Улучшение механизма поддержки выполнения требований (Игнатьев Денис, Евгений Олешкевич)

Что планируется сделать в последнюю очередь:
        1) Многопоточность
        2) Добавление новых примтивов, например, кривых Безье, сплайны
        3) Заливка фигур, работа с контурами
